# Design: Extract Benchmark Suite to Independent Repository

**Date:** 2026-03-26
**Status:** Approved
**Author:** zazabap

## Problem

The benchmark suite in `examples/benchmark/` has grown to ~1400 lines of code with 12 GB of data. It is a self-contained Julia project with its own `Project.toml` and has no coupling to the core library code. Keeping it in the main repo bloats clones and conflates two different change cadences.

## Decision Summary

- **New repo:** `zazabap/ParametricDFT-Benchmarks.jl`
- **History:** Preserved via `git filter-repo --subdirectory-filter`
- **Source branch:** `feature/multi-dataset-benchmark`
- **Parent repo integration:** Git submodule at `examples/benchmark`
- **Dependency:** ParametricDFT.jl via git URL (pinned to `main`)
- **Results:** Tracked in the new repo's git history
- **Data:** Remains gitignored (downloaded at runtime)

## Phase 1: Extract and Create New Repo

1. Clone `nzy1997/ParametricDFT.jl` to a temporary directory.
2. Checkout `feature/multi-dataset-benchmark`.
3. Run `git filter-repo --subdirectory-filter examples/benchmark` to rewrite history so that benchmark files are promoted to the repo root and only commits touching those files survive.
4. Create `zazabap/ParametricDFT-Benchmarks.jl` as a public repo on GitHub via `gh repo create`.
5. Add the new repo as remote and push the filtered history.

## Phase 2: Update Files in New Repo

### Project.toml

Change the ParametricDFT dependency from a relative path to a git URL:

```toml
[deps]
ParametricDFT = "..."

[sources]
ParametricDFT = {url = "https://github.com/nzy1997/ParametricDFT.jl", rev = "main"}
```

### .gitignore

Remove `results/` from the ignore list. Keep `data/` ignored.

### README.md

- Remove references to being a subdirectory of ParametricDFT.jl.
- Update project paths from `--project=examples/benchmark` to `--project=.`.
- Add a link back to `nzy1997/ParametricDFT.jl`.
- Update `run_all.sh` usage instructions.

### run_all.sh

Update internal paths from `examples/benchmark/` to `.` since scripts are now at the repo root.

### Run scripts

`run_quickdraw.jl`, `run_div2k.jl`, `run_clic.jl` use relative paths via `config.jl` and should work as-is. Verify during implementation.

### Results

Copy the existing `results/` directory from the current working tree into the new repo and commit. This includes:

- `results/smoke/`, `results/moderate/`, `results/heavy/` — per-preset metrics, trained models, plots
- `results/plots/` — cross-dataset plots
- `results/cross_dataset_summary.csv`, `results/timing_summary.csv`
- `results/*.log` — run logs

### Manifest.toml

Gitignore `Manifest.toml` since it is environment-specific and regenerated by `Pkg.instantiate()`.

## Phase 3: Submodule in Parent Repo

1. Remove `examples/benchmark/` directory from `ParametricDFT.jl` (on `feature/multi-dataset-benchmark`).
2. Add the new repo as a submodule:
   ```bash
   git submodule add https://github.com/zazabap/ParametricDFT-Benchmarks.jl examples/benchmark
   ```
3. Commit the `.gitmodules` file and submodule pointer.

After this, cloning with benchmarks:
```bash
git submodule update --init examples/benchmark
```

## Out of Scope

- No changes to `src/`, `test/`, or core library code.
- No changes to CLAUDE.md safety rules.
- No data files committed anywhere.
- No Julia General registry registration of ParametricDFT.jl.
